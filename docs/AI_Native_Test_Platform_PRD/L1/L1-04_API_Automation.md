## 1. 模块定位

API 自动化测试模块负责将 API 相关测试用例转化为可执行的 HTTP 测试脚本，支持 REST API 和 GraphQL 的功能验证、契约测试和链路测试。

**核心职责**：
- 定义 API 自动化测试的脚本规范（Pytest + httpx）
- 定义 AI 生成 API 测试脚本的输入输出约定
- 定义 API 测试的认证配置和环境管理
- 定义 API 测试的结果收集和断言规范

**不在本模块范围**：
- 性能/压力测试（V1.0 不纳入）
- Mock 服务器（V2.0 考虑）
- GraphQL Schema 验证（V2.0 考虑）

## 3. F4.1 API 脚本规范

### 3.1 脚本文件命名约定

```
tests/scripts/api/
├── test_<resource>_<case_id>.py   # 单用例脚本
├── conftest.py                     # 共享 fixtures（认证、client）
└── fixtures/                       # 共享请求/响应数据（可选）
    └── users_response.json
```

命名规则：
- 文件名：`test_` 前缀 + API 资源名（snake_case）+ `_` + 用例 ID（小写）
- 示例：`test_users_api_tc_101.py`、`test_orders_api_tc_215.py`

### 3.2 脚本文件结构标准

```python
"""
TC-ID: TC-101
Title: GET /api/users/{id} - 查询存在的用户
Module: user-api
Generated by: qoder test script
Generated at: 2026-04-12T10:00:00Z
"""
import pytest
import httpx


# ─── 测试数据（内联） ──────────────────────────────────────────────────────
EXISTING_USER_ID = 1
EXPECTED_USERNAME = "testuser"


# ─── 测试函数 ──────────────────────────────────────────────────────────────
def test_TC_101_get_existing_user(api_client: httpx.Client):
    """
    GET /api/users/{id} 返回指定用户的完整信息。

    前提条件：ID 为 1 的用户已存在
    """
    # Step 1: 发送 GET 请求
    response = api_client.get(f"/api/users/{EXISTING_USER_ID}")

    # Step 2: 验证状态码
    assert response.status_code == 200, \
        f"期望 200，实际 {response.status_code}，响应体：{response.text}"

    # Step 3: 验证响应结构
    data = response.json()
    assert "id" in data, "响应缺少 id 字段"
    assert "username" in data, "响应缺少 username 字段"
    assert "email" in data, "响应缺少 email 字段"

    # Step 4: 验证数据正确性
    assert data["id"] == EXISTING_USER_ID
    assert data["username"] == EXPECTED_USERNAME
```

### 3.3 强制规范

**必须遵守**：

1. **HTTP 客户端**：统一使用 `httpx.Client`（同步）或 `httpx.AsyncClient`（异步），不使用 `requests`

2. **断言规范**：
   - 每个断言必须有清晰的错误消息（`assert condition, "message"`）
   - 状态码断言必须是第一个断言
   - 断言顺序：状态码 → 响应结构 → 响应数据

3. **测试数据**：
   - 内联常量：适用于简单静态数据
   - `pytest.mark.parametrize`：适用于多组数据测试（如边界值）
   - 环境变量：适用于密钥、令牌等敏感数据

4. **响应验证深度**：
   - P0 接口：验证状态码 + 结构 + 关键字段值
   - P1 接口：验证状态码 + 关键字段
   - 不做全量响应比对（避免脆化）

5. **幂等性**：
   - GET/DELETE 测试天然幂等
   - POST/PUT/PATCH 测试：在 teardown 中清理创建的数据

### 3.4 conftest.py 标准模板

```python
"""
conftest.py - 由 qoder test script --init 生成，可手动修改
"""
import os
import pytest
import httpx


def pytest_addoption(parser):
    parser.addoption("--api-base-url", default="http://localhost:8000")
    parser.addoption("--api-key", default=None)
    parser.addoption("--auth-token", default=None)


@pytest.fixture(scope="session")
def api_base_url(pytestconfig):
    return pytestconfig.getoption("--api-base-url")


@pytest.fixture(scope="session")
def auth_headers(pytestconfig):
    """构建认证请求头，优先使用命令行参数，否则从环境变量读取。"""
    api_key = pytestconfig.getoption("--api-key") or os.environ.get("API_KEY")
    auth_token = pytestconfig.getoption("--auth-token") or os.environ.get("AUTH_TOKEN")

    headers = {}
    if api_key:
        headers["X-API-Key"] = api_key
    if auth_token:
        headers["Authorization"] = f"Bearer {auth_token}"
    return headers


@pytest.fixture(scope="session")
def api_client(api_base_url, auth_headers) -> httpx.Client:
    """会话级 HTTP 客户端，共享 TCP 连接。"""
    with httpx.Client(
        base_url=api_base_url,
        headers=auth_headers,
        timeout=30.0,
        follow_redirects=True,
    ) as client:
        yield client


@pytest.fixture
def api_client_no_auth(api_base_url) -> httpx.Client:
    """无认证的 HTTP 客户端，用于测试未授权场景。"""
    with httpx.Client(
        base_url=api_base_url,
        timeout=30.0,
    ) as client:
        yield client
```

## 4. F4.2 AI 生成 API 脚本

### 4.1 输入规范

CLI 命令：`qoder test script --case <case-id> [--spec <openapi-file>]`

**输入来源（按优先级）**：

1. **测试用例 Markdown 文件**（主要输入）
   - API 端点（Endpoint）
   - 请求方法（Method）
   - 请求头（Headers）
   - 请求体（Request Body）
   - 预期响应状态码
   - 预期响应字段

2. **OpenAPI 规范文件**（辅助输入，可选）
   - Schema 定义（请求/响应结构）
   - 安全方案（Security Schemes）
   - 参数约束

3. **知识库上下文**（辅助输入）
   - 工程内 `.md` 文件中的 API 文档
   - 历史脚本中的认证模式

4. **环境配置**（`tests/.qoder/config.toml`）

### 6.3 日志收集 Fixture

在 conftest.py 中通过 httpx 事件钩子实现自动日志记录：

```python
import httpx
import json
from datetime import datetime


class RequestLogger:
    def __init__(self, log_dir: str, test_name: str):
        self.log_dir = log_dir
        self.test_name = test_name
        self.entries = []

    def log_request(self, request: httpx.Request):
        entry = {
            "time": datetime.utcnow().isoformat() + "Z",
            "type": "request",
            "method": request.method,
            "url": str(request.url),
            "headers": self._sanitize_headers(dict(request.headers)),
        }
        if request.content:
            try:
                entry["body"] = json.loads(request.content)
            except Exception:
                entry["body"] = request.content.decode("utf-8", errors="replace")
        self.entries.append(entry)

    def log_response(self, response: httpx.Response):
        entry = {
            "type": "response",
            "status_code": response.status_code,
            "elapsed_ms": int(response.elapsed.total_seconds() * 1000),
            "headers": dict(response.headers),
        }
        try:
            entry["body"] = response.json()
        except Exception:
            entry["body"] = response.text
        self.entries.append(entry)

    def _sanitize_headers(self, headers: dict) -> dict:
        sensitive_keys = {"authorization", "x-api-key", "cookie"}
        return {
            k: "***（已脱敏）***" if k.lower() in sensitive_keys else v
            for k, v in headers.items()
        }

    def save(self):
        import os
        os.makedirs(self.log_dir, exist_ok=True)
        log_path = f"{self.log_dir}/{self.test_name}.txt"
        with open(log_path, "w", encoding="utf-8") as f:
            for entry in self.entries:
                f.write(json.dumps(entry, ensure_ascii=False, indent=2))
                f.write("\n---\n")
        return log_path
```

## 7. F4.5 链路测试（V1.5）

### 7.1 功能描述

链路测试（End-to-End API Chain）支持多步 API 请求，后续步骤依赖前序步骤的响应数据。

典型场景：
- 创建资源 → 查询资源 → 更新资源 → 删除资源（CRUD 完整链路）
- 登录 → 获取用户信息 → 执行业务操作 → 退出登录

### 7.2 链路测试脚本规范

```python
"""
TC-ID: TC-200
Title: 订单完整生命周期 - 创建、查询、取消
Module: orders-api
Type: chain
"""
import pytest
import httpx


def test_TC_200_order_lifecycle(api_client: httpx.Client):
    """订单完整生命周期：创建→查询→取消。"""
    # Step 1: 创建订单
    create_response = api_client.post("/api/orders", json={
        "product_id": 1,
        "quantity": 2,
    })
    assert create_response.status_code == 201, \
        f"创建订单失败：{create_response.text}"
    order_id = create_response.json()["id"]

    try:
        # Step 2: 查询订单
        get_response = api_client.get(f"/api/orders/{order_id}")
        assert get_response.status_code == 200
        order_data = get_response.json()
        assert order_data["status"] == "pending"

        # Step 3: 取消订单
        cancel_response = api_client.post(f"/api/orders/{order_id}/cancel")
        assert cancel_response.status_code == 200

        # Step 4: 验证状态变更
        final_response = api_client.get(f"/api/orders/{order_id}")
        assert final_response.json()["status"] == "cancelled"

    finally:
        # Teardown: 清理测试数据
        api_client.delete(f"/api/orders/{order_id}")
```

### 8.3 Schema 验证注入

当提供 `--spec` 参数时，生成的脚本自动包含 Schema 验证逻辑：

```python
import jsonschema
from pathlib import Path
import json
import yaml


def load_schema(spec_file: str, path: str, method: str, status_code: int) -> dict:
    """从 OpenAPI 文件中提取指定接口的响应 Schema。"""
    with open(spec_file) as f:
        spec = yaml.safe_load(f)
    
    response_schema = (
        spec["paths"][path][method.lower()]
        ["responses"][str(status_code)]
        ["content"]["application/json"]["schema"]
    )
    return response_schema


def test_TC_101_get_user_with_schema_validation(api_client, api_spec):
    """GET /api/users/{id} 响应符合 OpenAPI Schema 定义。"""
    response = api_client.get("/api/users/1")
    assert response.status_code == 200

    # Schema 验证
    schema = load_schema(api_spec, "/api/users/{id}", "GET", 200)
    try:
        jsonschema.validate(instance=response.json(), schema=schema)
    except jsonschema.ValidationError as e:
        pytest.fail(f"响应不符合 OpenAPI Schema：{e.message}")
```

## 12. 附录：典型脚本示例

### A. POST 创建资源

```python
"""
TC-ID: TC-150
Title: POST /api/users - 创建新用户成功
Module: user-api
"""
import pytest
import httpx


NEW_USER_DATA = {
    "username": "newuser_test",
    "email": "newuser_test@example.com",
    "password": "NewUser@123",
}
EXPECTED_FIELDS = {"id", "username", "email", "created_at"}


def test_TC_150_create_user_success(api_client: httpx.Client):
    """POST /api/users 创建新用户，返回 201 和用户对象。"""
    created_user_id = None
    try:
        # Step 1: 发送创建请求
        response = api_client.post("/api/users", json=NEW_USER_DATA)

        # Step 2: 验证状态码
        assert response.status_code == 201, \
            f"期望 201，实际 {response.status_code}，响应：{response.text}"

        # Step 3: 验证响应结构
        data = response.json()
        assert EXPECTED_FIELDS.issubset(data.keys()), \
            f"响应缺少字段：{EXPECTED_FIELDS - data.keys()}"

        # Step 4: 验证数据
        assert data["username"] == NEW_USER_DATA["username"]
        assert data["email"] == NEW_USER_DATA["email"]
        assert "password" not in data, "响应不应包含 password 字段"

        created_user_id = data["id"]

    finally:
        # Teardown: 删除测试用户
        if created_user_id:
            api_client.delete(f"/api/users/{created_user_id}")
```

### B. 权限验证场景

```python
"""
TC-ID: TC-151
Title: GET /api/admin/users - 无权限用户访问管理接口
Module: user-api
"""
import httpx


def test_TC_151_unauthorized_admin_access(api_client_no_auth: httpx.Client):
    """未认证用户访问管理接口，返回 401。"""
    response = api_client_no_auth.get("/api/admin/users")

    assert response.status_code == 401, \
        f"期望 401，实际 {response.status_code}"

    data = response.json()
    assert "error" in data or "message" in data, \
        "401 响应应包含 error 或 message 字段"
```

### C. 分页查询

```python
"""
TC-ID: TC-155
Title: GET /api/users - 分页查询用户列表
Module: user-api
"""
import httpx


def test_TC_155_list_users_pagination(api_client: httpx.Client):
    """分页查询：page=1&limit=10 返回最多 10 条记录和分页元数据。"""
    response = api_client.get("/api/users", params={"page": 1, "limit": 10})

    assert response.status_code == 200
    data = response.json()

    # 验证分页结构
    assert "items" in data, "响应缺少 items 字段"
    assert "total" in data, "响应缺少 total 字段"
    assert "page" in data, "响应缺少 page 字段"

    # 验证分页数据
    assert len(data["items"]) <= 10, "返回条数超过 limit=10"
    assert data["page"] == 1
    assert isinstance(data["total"], int) and data["total"] >= 0
```
